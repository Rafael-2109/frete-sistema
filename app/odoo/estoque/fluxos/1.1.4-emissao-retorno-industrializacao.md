**Quando:** automatizar a **SAÍDA** do retorno de industrialização (Etapa 4). O operador faz
**só o de hoje** na NF-1: cria o picking do PA, valida (→ nasce a NF-1 serviço **5124**, j847,

## Arquitetura (DECISÃO SOT §6.3 — Opção B)

A montagem da NF-2 (op **2864** + **recompute** `onchange_l10n_br_calcular_imposto` + remap conta
fp **111** → 1150100012) e a transmissão SEFAZ são **irredutivelmente server-side** (o recompute
só persiste dentro de uma server action — `s48`/`s49`; ver `PROTECAO §N34`). Por isso:

| Camada | Onde | O quê |
|---|---|---|
| **Gatilho** | `ir.cron`-Odoo (padrão dos robôs) | G1: NF-1 j847 sem NF-2 vinculada · G2: NF-1 `cstat=100` com NF-2 não transmitida |
| **Execução** | **server action persistente** (infra-Odoo, `app/odoo/estoque/provisioning/`) | G1 body monta+posta a NF-2 (genealogia server-side) · G2 body transmite (`s54`) |
| **Lado nosso (READ)** | orchestrator C3 `saida_retorno_industrializacao.py` | `planejar` (spec via Descoberta = ORÁCULO) · `validar` (NF-2 da SA vs spec) · `medir` (baixa PASSIVA pelo ciclo) |
| **Resiliência** | `provisioning/` | provisionador idempotente (re-cria SA+cron por `name`) + monitor anti-upgrade (checagem D8 + hook SessionStart) |

> **Por que NÃO há átomo XML-RPC da montagem da saída:** seria *lying parameter* (o recompute não
> persiste via XML-RPC). `montar_invoice_entrada_direta` fica PURO para a ENTRADA. Ver `PROTECAO §N34`.

## Sequência (regime de produção)

```
0) Operador: cria picking do PA + valida (NF-1 5124 nasce em j847) + TRANSMITE a NF-1   [atuação humana — única]
1) cron G1 detecta NF-1 j847 sem NF-2 → SA G1 (server-side):
     genealogia (lote PA → MO/sub-MO → materiais de terceiros 31092, exclui água, rateio,
       preço via SVL-entrada) → monta NF-2 (RETIND 1083, op 2864, calcular_imposto=False) →
       recompute → remap conta fp 111 → R3 (refNFe remessa + invoice_origin) → action_post (baixa PASSIVA)
2) cron G2 detecta NF-1 cstat=100 + NF-2 posted não transmitida → SA G2 (server-side):
     cross-refNFe (chave NF-1 na NF-2) → action_previsualizar_xml_nfe + action_gerar_nfe → SEFAZ  [autorizado pela transmissão da NF-1]
3) (nosso, canary/auditoria) saida_retorno_industrializacao: validar(nf1,nf2) vs spec + medir baixa PASSIVA
```

## Gate de sucesso (por CICLO, não saldo global)
NF-2: journal RETIND 1083 · n_linhas == nº componentes da Descoberta · CFOP `5902` · CST `50` ·
conta linha `1150100012` · total ≈ spec (rateio) · R3 (`invoice_origin` + refNFe remessa) ·
**baixa PASSIVA `5101020001` (conta 26667) = total insumos** (`s40` mediu Δ+279,23 no piloto lote inteiro).
Medição: `python -m app.odoo.estoque.orchestrators.saida_retorno_industrializacao validar|medir`.
