Mapeamento Modelo HORA → Produto TagPlus

Checklist
Cada modelo de moto da HORA mapeia para dois identificadores no TagPlus:
  • ID TagPlus (inteiro): usado na emissão de NFe (campo id em GET /produtos). O TagPlus rejeita códigos string aqui com erro 400 — só ID numérico funciona.
  • Código de Produto (DANFE): o código curto que aparece na coluna CÓDIGO da DANFE PDF emitida pelo TagPlus (ex: MT-JETMAX, MT-X12, BK-VTB4). Usado no backfill de NFs já emitidas — o parser DANFE lê esse código e resolve modelo_id deterministicamente, sem depender de LLM adivinhar o nome do modelo.
CFOP padrão: 5.403 (intra, venda de mercadoria com ST — contribuinte substituído) — o builder ajusta para 6.403 quando UF do cliente difere da UF do emitente.
