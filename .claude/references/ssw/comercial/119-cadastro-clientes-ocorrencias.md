# Opção 119 — Cadastro de Clientes - Ocorrências

> **Módulo**: Comercial
> **Referência interna**: Opção 385
> **Páginas de ajuda**: 1 página consolidada
> **Atualizado em**: 2026-02-15

## Função

Registra e armazena ocorrências no relacionamento com o cliente, mantendo histórico cronológico de eventos manuais e automáticos.

## Quando Usar

- Registrar manualmente ocorrências importantes do relacionamento comercial
- Consultar histórico de eventos do cliente (visitas, reajustes, negociações)
- Documentar comunicações e acordos com clientes
- Acompanhar relatórios de visitas de vendedores

## Campos / Interface

### Inserir Ocorrências

- **3 linhas de texto**: Espaço para registrar a ocorrência que será gravada no histórico do cliente

### Tabela de Histórico

Exibe todas as ocorrências em ordem cronológica, incluindo:
- Ocorrências geradas **manualmente** (inseridas pelo usuário)
- Ocorrências geradas **automaticamente** pelo sistema (ex: reajustes de tabelas)

## Integração com Outras Opções

- **Relatórios de visitas de vendedores**: Gerados automaticamente pela opção 119 e registrados no histórico
- **Reajustes de tabelas**: Sistema registra automaticamente quando há alteração de preços para o cliente

## Observações e Gotchas

### Armazenamento Cronológico

Todas as ocorrências são armazenadas em ordem cronológica, facilitando rastreamento temporal do relacionamento com o cliente.

### Ocorrências Automáticas vs Manuais

O sistema diferencia entre:
- **Manuais**: Inseridas pelo usuário nas 3 linhas de texto
- **Automáticas**: Geradas por processos do sistema (reajustes, visitas)

Ambas ficam no mesmo histórico cronológico.

### Limite de Caracteres

As 3 linhas permitem registro conciso da ocorrência. Para detalhamentos maiores, considerar múltiplas inserções ou documentação externa.
